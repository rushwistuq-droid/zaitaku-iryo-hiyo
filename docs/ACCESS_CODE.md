# アクセスコード（有料note限定公開）運用ガイド

本ツールは、トップに**アクセスコード入力ゲート**を設けており、正しいコードを入力しないと使えません。コードは**有料note内でのみ配布**する運用を想定しています。

---

## 現在のアクセスコード

> ⚠️ **平文コードはこのリポジトリに保存していません。**
> このリポジトリは公開されているため、コードをファイルに書くと第三者に読まれてゲートが無意味になります。
> そのため `app.js` には **ハッシュのみ** を置き、平文コードは**あなたの手元（＝有料note本文）だけ**で管理します。

- 現在有効なコードに対応するハッシュ（`app.js` の `ACCESS_CODE_HASH`）：
  ```
  bb7a912e5bdc667bc5c437ed0d09b14d529911f705c22bdd0fff264962d5e433
  ```
- 入力は**大文字小文字を区別しません**（前後の空白は自動で除去）。
- 一度正しく入力すると、その端末・ブラウザに記憶され（`localStorage`）、次回以降はゲートをスキップします。
- **設定した平文コードは、有料noteの本文（有料エリア）と、あなた自身の控え（パスワード管理ツール等）に保管**してください。万一忘れても、下記手順で新しいコードに作り直せます（ハッシュから平文は復元できません）。

---

## 仕組み

- 平文コードはソースコードに置いていません。**SHA-256 ハッシュ**だけを `app.js` に埋め込み、入力値のハッシュと照合します。
- そのため「ブラウザでソースを表示しただけ」ではコードは分かりません。
- 該当箇所は `app.js` の先頭 `ACCESS_CODE_HASH` 定数、UIは `index.html` の `#access-gate`、装飾は `styles.css` の「アクセスコードゲート」節です。

### ⚠️ セキュリティについて（正直な但し書き）

これは**一般の購入者に対する抑止策**であり、完全な保護（DRM）ではありません。

- JavaScript を読み解ける人がハッシュを総当たり・解析すれば突破は可能です（コードを推測しにくい文字列にすると耐性は上がります）。
- 完全な保護には**サーバー側の認証**が必要で、それは静的サイト（GitHub Pages）の範囲外です。
- 数百円のnoteに対する「気軽なタダ使いの防止」としては十分実用的で、多くのnote販売者が採用する方式です。

## コードを変更する方法

第三者にコードが出回った場合や、定期的に変えたい場合は、次の手順で差し替えます。

### 1. 新しいコードのハッシュを生成

ローカルに Node.js があれば、以下で生成できます（`新しいコード` を任意の文字列に置き換え）。

```bash
node -e "console.log(require('crypto').createHash('sha256').update('新しいコード'.trim().toUpperCase()).digest('hex'))"
```

出力された64文字の16進文字列がハッシュです。

### 2. `app.js` のハッシュを差し替え

`app.js` 先頭付近の次の行を、生成したハッシュに置き換えます。

```javascript
const ACCESS_CODE_HASH = '（ここを新しいハッシュに）';
```

### 3. 本ファイルの「現在のアクセスコード」も更新し、commit → push

`main` にマージすると GitHub Pages が数分で更新されます。

> 💡 コードを変えると `ACCESS_CODE_HASH` も変わるため、**既存購入者の記憶（localStorage）も自動的に無効化**され、再入力が必要になります。旧コードを一斉に失効させたいときに有効です。
